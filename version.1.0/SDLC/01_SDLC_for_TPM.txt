# SDLC for Technical Project Managers

## Learning Goal

Understand how a Technical Project Manager manages a software project from idea to production.

---

# What is SDLC?

SDLC (Software Development Life Cycle) is a structured process used to build software in a predictable, controlled, and high-quality manner.

It defines:

- How software is planned
- How requirements are collected
- How development happens
- How testing is performed
- How releases are managed
- How maintenance continues after launch

As a TPM, your responsibility is to ensure every phase is executed successfully.

---

# Standard SDLC Phases

1. Requirement Gathering & Analysis

	Objective: Understand business needs.

	Activities:

		Gather functional and non-functional requirements
		Conduct stakeholder meetings
		Define project scope
		Prepare requirement documents (BRD/PRD)

	Output:

		Approved requirements
		Product backlog or requirement specification



2. Planning
3. Design
4. Development
5. Testing
6. Deployment
7. Maintenance

---

# Phase 1 — Requirement Gathering

Objective:
Understand what the customer wants.

Activities

- Meet stakeholders
- Gather business requirements
- Clarify scope
- Identify assumptions
- Identify dependencies
- Record functional requirements
- Record non-functional requirements

TPM Responsibilities

✔ Schedule meetings

✔ Ask clarification questions

✔ Capture requirements

✔ Prepare BRD

✔ Prepare user stories

✔ Identify risks

Deliverables

- BRD
- User Stories
- Acceptance Criteria
- Requirement Traceability Matrix

---

# Phase 2 — Planning

Objective

Plan the entire project.

Activities

- Effort estimation
- Resource planning
- Timeline planning
- Sprint planning
- Cost estimation
- Risk planning

TPM Responsibilities

✔ Capacity planning

✔ Timeline creation

✔ Roadmap preparation

✔ Budget estimation

✔ Team allocation

✔ Stakeholder communication

Deliverables

- Project Plan
- Roadmap
- Sprint Plan
- Risk Register
- Resource Plan

# Phase 3 — Design

Design how the solution will work.

Activities

- UX Design
- Database Design
- API Design
- Architecture Design

TPM Responsibilities

✔ Coordinate teams

✔ Resolve dependencies

✔ Review design progress

✔ Ensure design approval

Activities

✔ Sprint tracking

✔ Remove blockers

✔ Daily monitoring

✔ Capacity adjustments

✔ Jira updates

Deliverables

- Build
- Sprint Report

---

# Phase 5 — Testing

Objective

Activities

- Functional Testing
- Regression Testing

TPM Responsibilities

✔ Defect tracking

✔ Prioritize bugs

✔ Coordinate QA

✔ Review test reports
