# EDTECH DATA MIGRATION PROJECT PLAN

## Project Name

Legacy Student Information System (SIS) to Cloud-Based Education Platform

## Domain

Education Technology (EdTech)

## Project Duration

7 Months

## Methodology

Agile Scrum

## Cloud Platform

AWS

## Project Objective

Migrate student, faculty, academic, and financial data from the legacy Student Information System (SIS) to a cloud-based Learning Management System (LMS) and Education CRM while ensuring high data accuracy, minimal downtime, and secure data transfer.

---

## BUSINESS PROBLEM

The existing Student Information System had several challenges:

* Manual data management
* Duplicate student records
* Slow report generation
* Limited scalability
* Poor integration with LMS and CRM
* High infrastructure maintenance cost
* Limited support for online learning

The institution decided to modernize its education platform through a cloud-based digital transformation initiative.

## TPM INTERVIEW SUMMARY

I managed a 7-month data migration program to modernize a legacy Student Information System by migrating student, faculty, admissions, attendance, examination, and fee data to a cloud-based Learning Management System and Education CRM on AWS. I coordinated business analysts, solution architects, ETL developers, DBAs, QA, DevOps, and academic stakeholders. I was responsible for project planning, sprint execution, stakeholder communication, RAID management, risk mitigation, dependency tracking, mock migrations, production cutover, and Hypercare support. The project achieved 99.9% data accuracy, minimal downtime, zero critical production issues, and a successful transition to the new platform.
