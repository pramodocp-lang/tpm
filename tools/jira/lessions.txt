JIRA FOR TPM - LESSON 1
WHAT IS JIRA?
========================================

Definition:
Jira is a project management and issue tracking tool used by software teams
to plan, track, and deliver work.

Why Companies Use Jira:
- Track project progress
- Manage Agile/Scrum processes
- Assign work to team members
- Track bugs and issues
- Monitor sprint execution
- Generate reports and dashboards

----------------------------------------
REAL TPM EXAMPLE
----------------------------------------

Requirement:
"Student should be able to book a counseling session."

In Jira:

Epic
└── Student Counseling Module

    Story
    └── Student can book counseling session

        Task
        ├── Design UI
        ├── Develop API
        ├── Create Database Table
        └── Testing

----------------------------------------
COMMON JIRA TERMS
----------------------------------------

Project   = Collection of all project work
Issue     = Any work item in Jira
Epic      = Large feature or initiative
Story     = User requirement
Task      = Technical work item
Bug       = Defect or issue
Sub-task  = Smaller piece of a task
Sprint    = Time-boxed work cycle
Backlog   = List of future work
Board     = Visual workflow view

----------------------------------------
