=========================================================
PROJECT TRACKING FOR TPM
=========================================================

1. WHAT IS PROJECT TRACKING?

Project Tracking is the process of monitoring project
progress against the planned schedule, scope, and goals.

The objective is to ensure that the project is delivered:

- On Time
- Within Budget
- With Expected Quality

---------------------------------------------------------

2. WHY IS PROJECT TRACKING IMPORTANT?

Project tracking helps a TPM:

- Monitor progress
- Identify delays early
- Track dependencies
- Manage risks and issues
- Improve team accountability
- Provide stakeholder visibility

---------------------------------------------------------

3. PROJECT TRACKING COMPONENTS

A. Tasks
Individual work items.

Examples:
- Design Login Page
- Develop Login API
- Test Login Feature

B. Owner
Person responsible for completing the task.

Examples:
- Developer
- QA Engineer
- Designer

C. Start Date
Date when work begins.

D. End Date
Expected completion date.

E. Status
Current progress of the task.

Examples:
- Not Started
- In Progress
- Blocked
- Completed

---------------------------------------------------------

---------------------------------------------------------

5. STATUS DEFINITIONS

Not Started
- Work has not begun.

In Progress
- Work is actively being performed.

Blocked
- Progress cannot continue due to dependency or issue.

Completed
- Task has been finished successfully.

---------------------------------------------------------

6. RED FLAGS TO TRACK

- Missed deadlines
- Blocked tasks
- Resource shortages
- Scope changes
- High-risk items
- Unresolved defects

---------------------------------------------------------

7. WEEKLY TPM REVIEW

Review the following:

9. TPM INTERVIEW QUESTION

Q: How do you track project progress?

Answer:

I track project progress using project trackers and Agile
tools such as Jira. I monitor task completion, sprint
progress, risks, issues, dependencies, and resource
utilization. I regularly review project status with the
team and provide updates to stakeholders through reports
and dashboards.

---------------------------------------------------------

10. KEY METRICS

- % Completion
- Tasks Completed
- Tasks Delayed
- Open Risks
- Open Issues
- Sprint Velocity
- Defect Count
- Resource Utilization
