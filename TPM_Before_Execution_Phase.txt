=========================================================
TPM BEFORE EXECUTION PHASE (FINAL READINESS STAGE)
=========================================================

1. WHAT IS BEFORE EXECUTION PHASE?

This is the final stage before development starts.

Goal:
- Ensure project is fully ready for execution
- Remove ambiguity
- Confirm team readiness
- Prevent future blockers

=========================================================

2. TPM ROLE BEFORE EXECUTION

TPM ensures everything is READY to start work smoothly.

Key responsibilities:

1. Final scope confirmation
2. Sprint readiness check
3. Dependency validation
4. Resource confirmation
5. Risk review
6. Tool setup (Jira, Confluence, etc.)
7. Stakeholder alignment
8. Kickoff preparation

=========================================================

3. REQUIREMENT FREEZE

TPM ensures requirements are stable:

- No major changes expected
- All features clearly defined
- Acceptance criteria approved

Output:
- Signed-off requirements

=========================================================

4. SCOPE FINALIZATION

TPM confirms:

IN SCOPE:
- Features approved for development

OUT OF SCOPE:
- Items excluded from current release

Goal:
Avoid scope creep during execution

=========================================================

5. SPRINT READINESS CHECK

TPM verifies:

- Stories are well defined
- Tasks are estimated
- Dependencies are clear
- Team capacity is known

If not ready:
- Send back for refinement

=========================================================

6. DEPENDENCY FINAL CHECK

TPM ensures:

- External teams are aligned
- APIs are ready or scheduled
- Design assets are completed
- QA environment is available

=========================================================

7. RESOURCE CONFIRMATION

TPM checks:

- Developers assigned
- QA assigned
- Designers assigned
- DevOps ready

Goal:
No resource gap during sprint

=========================================================

8. RISK REVIEW (FINAL CHECK)

TPM reviews known risks:

Examples:
- Delay in API delivery
- Resource unavailability
- Environment instability

Actions:
- Mitigation plan prepared
- Backup plan defined

=========================================================

9. TOOL SETUP

TPM ensures tools are ready:

- Jira board created
- Epics and stories added
- Confluence pages ready
- Dashboards configured

=========================================================

10. KICKOFF MEETING

TPM conducts kickoff meeting:

Attendees:
- Product Owner
- Developers
- QA
- Business Stakeholders

Agenda:
- Project overview
- Sprint goals
- Roles & responsibilities
- Timeline
- Risks and dependencies

=========================================================

11. OUTPUT OF BEFORE EXECUTION PHASE

Final deliverables:

- Ready backlog
- Sprint plan approved
- Resources confirmed
- Risks documented
- Dependencies aligned
- Stakeholders aligned
- Execution kickoff completed

=========================================================

12. TPM INTERVIEW ANSWER

Q: What do you do before execution starts?

Answer:

Before execution starts, I ensure that all requirements
are finalized, scope is clearly defined, and the backlog
is ready. I validate dependencies, confirm resource
availability, review risks, and ensure tools like Jira
and Confluence are set up. I also conduct a kickoff
meeting with all stakeholders to align on goals and
ensure the team is fully ready for sprint execution.

=========================================================

13. ONE-LINE SUMMARY

Before execution, TPM ensures requirements, scope,
resources, dependencies, and tools are fully ready
for smooth project delivery.
