SDLC (Software Development Life Cycle) 1. Requirement Gathering - Understand user/business needs - Define scope - Output: SRS document 2. Planning - Timeline, cost, resources - Risk analysis - Technology selection - Output: Project Plan 3. System Design - HLD (Architecture design) - LLD (Database, APIs) - Output: Design documents 4. Development - Coding based on design - Follow standards - Use Git/version control 5. Testing - Unit, Integration, System, UAT - Bug reporting and fixing 6. Deployment - Release to production - Setup servers/cloud - Monitor system 7. Maintenance - Bug fixes - Enhancements - Performance improvements FLOW: Requirements → Planning → Design → Development → Testing → Deployment → Maintenance
